This file provides guidance to Codex (Codex.ai/code) when working with code in this repository.

## Project Overview

This is HUTAO's Tech Blog, an Astro 5 static site deployed to GitHub Pages. It uses MDX for blog posts, TailwindCSS for styling, React for optional interactive components, and Expressive Code for code highlighting.

## Common Commands

Use pnpm. The GitHub Pages workflow builds with `pnpm@10.28.2` and Node 20, so treat that pair as the deployment baseline even if local Node/pnpm versions differ.

```bash
pnpm install        # install dependencies
pnpm dev            # start Astro dev server at http://localhost:4321
pnpm build          # build the static site
pnpm preview        # preview the production build
pnpm new            # run create-post.js to scaffold a new MDX post
pnpm astro --help   # run Astro CLI commands
```

There is no dedicated lint or test script in `package.json` at the time of writing. For validation, run `pnpm build`; it exercises Astro content collection schema validation and static page generation.

The repo uses `pnpm-lock.yaml` only. Do not add `package-lock.json`; prefer pnpm because the scripts, lockfile used by deployment, and workflow are pnpm-based.

## Architecture Notes

- `astro.config.mjs` configures the site URL (`https://awfp1314.github.io/`), Astro integrations (`@astrojs/tailwind`, `astro-expressive-code`, `@astrojs/mdx`, `@astrojs/react`), dev server port `4321`, `trailingSlash: "ignore"`, and `public/` passthrough assets.
- `src/content.config.ts` defines the single `blog` content collection. Blog files are loaded from `src/blog/**/*.mdx` and require frontmatter fields: `title`, `pubDate`, `description`, `author`, and `tags`; optional/defaulted fields include `image`, `lang` (`zh` by default), and `alternateSlug`.
- Blog post routes are generated by `src/pages/[...slug].astro` from the `blog` collection. The route uses each post's content ID, so the nested path under `src/blog/` becomes the URL path.
- `src/layouts/MainLayout.astro` is the shared page shell. It imports global CSS, renders `Header` and `Footer`, initializes light/dark theme state, and manages client-side language switching through `localStorage`, custom events, and `location.replace()` so language toggles do not add browser history entries.
- `src/layouts/MarkdownPostLayout.astro` wraps rendered MDX posts, displays frontmatter metadata, and handles article-language switching via the post's `alternateSlug`.
- `src/pages/blog/index.astro` builds the Chinese archive from `getCollection("blog")`, sorts posts by `pubDate` descending, and renders `BlogArchive`.
- `src/pages/blog-en.astro` builds the English archive from `getCollection("blog")`, sorts posts by `pubDate` descending, and renders `BlogArchive`.
- `src/components/BlogArchive.astro` supports archive search and preserves scroll/search state with `sessionStorage` when opening a post.
- Homepage, about pages, and blog archives are duplicated by language (`index.astro` / `index-en.astro`, `about.astro` / `about-en.astro`, `blog/index.astro` / `blog-en.astro`).
- Shared navigation and social link constants live in `src/consts.js`. `src/i18n.js` contains translation helpers, but much of the current UI language behavior is implemented inline in Astro page/layout scripts.
- Static assets and `.nojekyll` live under `public/`. Reference public assets from MDX with root-relative paths such as `/blog-images/...` when adding blog images.
- Tailwind is configured in `tailwind.config.mjs`, uses class-based dark mode, scans `src/**/*`, and includes `@tailwindcss/typography`.

## Blog Content Conventions

- Place posts under `src/blog/<year>/<month>/<filename>.mdx`.
- Include `pubDate` as a `YYYY-MM-DD` string; the content schema requires it and validates it as a date.
- Use `lang: zh` or `lang: en` so the archive language filter can include the post correctly. If omitted, the content schema defaults to `zh`.
- For translated post pairs, set `alternateSlug` to the other post's collection ID/path without a leading slash, matching the path generated from `src/blog/`.
- `create-post.js` scaffolds Chinese-oriented frontmatter and content in the current year/month directory; verify generated frontmatter against `src/content.config.ts` before building.
- Several source files contain Chinese UI/content strings. Preserve UTF-8 when editing them; if terminal output displays mojibake, inspect the file bytes/editor encoding before rewriting text.

## Deployment

`.github/workflows/deploy.yml` deploys pushes to `main` through GitHub Pages using `withastro/action@v3`, Node 20, and `pnpm@10.28.2`.
